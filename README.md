# Real-time Anomaly Detection Alerts (IQR-based Monitoring)

Проект представляет собой автоматизированную систему мониторинга ключевых метрик приложения в реальном времени. Каждые 15 минут система проверяет данные из ClickHouse, выявляет аномалии с помощью метода межквартильного размаха (IQR) и отправляет оповещения в Telegram с визуализацией и ссылками на дашборды Superset.

## 🎯 Цели проекта

- **Оперативное обнаружение аномалий:** Выявлять резкие падения или всплески метрик раньше, чем это сделают пользователи или заказчики.
- **Минимизация времени инцидента:** Предоставлять аналитикам и разработчикам всю необходимую информацию для быстрого расследования (график, ссылка на дашборд).
- **Автоматизация мониторинга:** Полностью исключить ручной контроль метрик, доверив эту задачу Airflow и статистическим методам.
- **Масштабируемость:** Легко добавлять новые метрики в систему мониторинга.

## 📊 Набор данных и метрики

Система получает данные из двух таблиц ClickHouse каждые 15 минут:

- **`feed_actions`** (Лента новостей):
    - Активные пользователи в ленте (`dau_feed`)
    - Просмотры (`views`)
    - Лайки (`likes`)
    - CTR (`likes / views`)
- **`message_actions`** (Мессенджер):
    - Активные пользователи в мессенджере (`dau_messenger`)
    - Количество отправленных сообщений (`messages_sent`)

## 🔍 Алгоритм обнаружения аномалий (IQR)

Для каждой метрики используется метод **межквартильного размаха (Interquartile Range)**:

1.  **Сбор истории:** За последние `N` 15-минутных интервалов (например, 24 часа = 96 точек) собираются значения метрики.
2.  **Расчет границ:**
    - Вычисляется первый квартиль (Q1) — 25-й перцентиль.
    - Вычисляется третий квартиль (Q3) — 75-й перцентиль.
    - Рассчитывается межквартильный размах: `IQR = Q3 - Q1`
    - Определяются верхняя и нижняя границы нормы:
        - `Нижняя граница = Q1 - 1.5 * IQR`
        - `Верхняя граница = Q3 + 1.5 * IQR`
3.  **Сравнение:** Текущее значение метрики сравнивается с границами.
4.  **Алерт:** Если значение выходит за пределы `[нижняя граница, верхняя граница]`, фиксируется аномалия.


## 🤖 Формат Telegram-алерта

При обнаружении аномалии система отправляет в Telegram-чат сообщение, содержащее:

1.  **Текстовое описание аномалии:**
    - `Метрика:` Название метрики (users_feed, views, likes, ctr, users_message, count_message)
    - `Текущее значение:` Значение метрики в аномальном 15-минутном интервале
    - `Отклонение:` Процент изменения относительно предыдущего 15-минутного значения
    - `Ссылка на дашборд Superset:` https://superset.lab.karpov.courses/superset/dashboard/8239/ — для детального анализа всех метрик

2.  **Визуализация (график):**
    - Вложенное изображение (PNG) с тремя линиями:
        - `metric` — фактическое значение метрики за последние N интервалов
        - `up` — верхняя граница IQR (Q3 + 1.5 * IQR)
        - `low` — нижняя граница IQR (Q1 - 1.5 * IQR)
    - Ось X — временные метки (с разряженными подписями для читаемости)
    - Ось Y — значение метрики (от 0 до максимума)

**Пример сообщения:**
```
Метрика views:
текущее значение = 15420.00
отклонение от предыдущего значения +23.45%
Графики с основными метриками: https://superset.lab.karpov.courses/superset/dashboard/8239/
```

## 🔧 Используемые технологии и библиотеки

- **Язык:** Python 3.8+
- **Оркестрация:** Apache Airflow (`@dag`, `@task` декораторы)
- **База данных:** ClickHouse (библиотека `pandahouse`)
- **Анализ данных:** `pandas`, `numpy`
- **Визуализация:** `seaborn`, `matplotlib`
- **Уведомления:** Telegram Bot API (библиотека `python-telegram-bot` или `requests`)
- **Дополнительно:** `datetime` (для работы с временными метками)



## 🚀 Результат

Система работает круглосуточно и гарантирует, что команда узнает о проблеме не позже, чем через 15 минут после её возникновения. Вместо сухих цифр, алерт содержит:

- Контекст (график с историей)
- Статистически обоснованную границу нормы (IQR)
- Ссылку для глубокого погружения (Superset)

Это позволяет дежурному аналитику или разработчику быстро принять решение: это сбой в данных, реальная проблема или просто плановая нагрузка.
